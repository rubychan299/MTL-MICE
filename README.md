# MTL-MICE: Multi-Task Learning in Multiple Imputation by Chained Equation

This repository contains the public reproducibility materials for the paper "MTL-MICE: Multi-Task Learning in Multiple Imputation by Chained Equation" by Yuyu Chen and Yang Feng.

## Repository structure

- `manuscript/`: LaTeX source for the main paper and supplement, bibliography
  files, and copies of manuscript figures.
- `data/`: Public U.S. election case-study data and data-preparation scripts.
- `code/`: Core MTL-MICE implementation, simulation scripts, real-data analysis
  scripts, and a small toy example.
- `output/`: Final PDF figures produced for the manuscript and supplement.

## Software environment

The analyses were developed in R. The original large-scale runs were executed on
an HPC cluster with R 4.5.1 and SLURM job arrays. The repository does not
include an `renv` lockfile, so packages need to be installed manually.

Main R packages used across the repository:

- `caret`
- `doParallel`
- `dplyr`
- `foreach`
- `ggbreak`

## Quick start

From the repository root:

```r
install.packages(c(
  "caret", "doParallel", "dplyr", "foreach", "ggbreak", "ggplot2",
  "ggraph", "glmnet", "glmtrans", "igraph", "mice", "patchwork",
  "pROC", "readr", "tidygraph", "tidyr", "tidyverse"
))
```

The smallest self-contained example in this repository is the toy vignette:

```bash
quarto render code/MTL_MICE_toy_example.qmd
```

## Reproducing the manuscript components

### Core implementation

The main MTL-MICE code lives in `code/R/`. In particular, `code/R/mtlmice.R`
defines the imputation engine, while the remaining files provide
initialization, simulation, completion, and evaluation utilities.

### Simulation study

Simulation scripts are organized by scenario in `code/sim/`:

- `code/sim/k_sim/`: tasks-quantity-effect simulations.
- `code/sim/h_sim/`: transfer-strength simulations.
- `code/sim/h_mixed/`: mixed-outcome simulations.
- `code/sim/h_outlier/`: outlier-task simulations.
- `code/sim/summ/`: summary and plotting scripts for the simulation outputs.

The `.sh` files in these folders are the original SLURM submission wrappers
used for long batch runs. The repository includes the final figure PDFs in
`output/`, but it does not include the large intermediate `.RData` result files
generated by the full simulation sweeps.

### Real-data analysis

The county-level U.S. election case study uses the public files in
`data/uselection/`. The main components are:

- `data/data_cleaning_uselection.R`: constructs the state-level task objects and
  missingness replicates.
- `data/uselection/bystate_2020/uselect_list2020.RData`: processed state-level
  task list used by the manuscript's 2020 case study.
- `code/rda/rda_uselection_0.1.R` through `code/rda/rda_uselection_0.5.R`:
  batch scripts for missingness rates from 0.1 to 0.5.
- `code/rda/plot_rda.R`: summarises the real-data runs and generates the
  performance figures.
- `code/rda/extract_id_rda.R`: extracts and visualises state-to-state
  transferability networks.


## Manuscript files

The main paper source is `manuscript/mtlmice.tex`, with
`manuscript/supplement.tex` for the supplement. Figure copies used by the LaTeX
files are stored in `manuscript/plots/`.

## Notes

- Run scripts from the repository root unless a script explicitly documents a
  different working directory.
- Some long-running scripts preserve the original SLURM-oriented workflow used
  during development. For the public repository, the important final artifacts
  are the manuscript sources, the public data, the method implementation, and
  the exported figures under `output/`.
